# 辅种查看 (CrossSeedView)

跨下载器扫描种子，按 **种子名 + 大小** 识别辅种关系，用可折叠卡片列表展示辅种数量、保存路径与明细。支持交互式筛选、批量删种、以及与媒体库软/硬链接的联动清理。

## 主要功能

- **跨下载器识别辅种**：同一时刻扫描所有已配置下载器，按 `name + size` 分组，一眼看出哪些资源做了辅种、哪些是孤种。
- **交互式筛选（可保存）**：名称/路径关键词、大小区间、辅种数量上下限、下载器筛选；筛选条件可持久化到插件配置。
- **常见目录快捷入口**：自动提取 Top 8 保存路径的公共前缀，点击 chip 一键筛选该目录。
- **明细路径操作**：详情页可复制/高亮/一键"筛选此路径"。
- **可选删除功能**（`allow_delete` 关闭时按钮不显示）：
  - 单行/组头三态勾选 + 批量勾选。
  - **仅删种**：只从下载器移除任务，保留文件。
  - **删种+文件+清库**：移除任务、删除源文件，并联动清理 TransferHistory 记录的媒体库软/硬链接、同刮削目录内残留的 `.nfo`/`.jpg`/字幕，清理 DownloadHistory 并发送 `DownloadFileDeleted` 事件。
- **删种通知**：可选开启，删种成功/失败均通过系统通知渠道汇总，附种子名与清理路径预览。
- **定时刷新**：默认每日 04:00 全量扫描，也可从详情页点"立即重新扫描"。

## 配置项

| 项 | 说明 |
|---|---|
| CRON | 定时刷新表达式，默认 `0 4 * * *` |
| refresh_on_init | MP 启动时是否立即扫描一次 |
| include_all_tags | 是否包含 MP 内建标签之外的全部种子 |
| min_count / max_count | 辅种数量下限/上限 |
| downloader_filter / name_keyword / path_keyword | 常用筛选缺省值 |
| size_min_gb / size_max_gb | 大小区间（GB） |
| allow_delete | **启用删除按钮**，默认关闭。开启后详情页出现删除按钮 |
| notify | 删种时发送通知，默认关闭 |

## 使用注意

- **`allow_delete` 有风险**：一旦启用，明细页每一条辅种都会出现红色删除按钮。误点"删种+文件+清库"会删除源文件并清理媒体库链接，不可撤销。
- **PageRender 鉴权**：详情页所有 API 都通过 `?apikey=` 参数走 apikey 分支，跟 MP 前端 axios 拦截器无关。
- **失联时的兜底策略**：媒体库软链接被外部删除后，插件仍会尝试清空刮削目录残留元数据；`TransferHistory` 三级回退（主 hash → 同组辅种 hash → content_path）覆盖辅种场景下的清理。
